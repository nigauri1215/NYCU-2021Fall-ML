Model Construction & Validation

此次作業都使用sklearn提供的model
一些說明如下

  1. Decision Tree
    • ID3 (Iterative Dichotomiser 3) ，該算法創建了一個 multiway tree，為每個節點（即以貪婪的方式）找到將為分類目標產生最大信息增益的分類特徵。樹生長到它們的最大大小，然後通常應用修剪步驟來提高樹泛化到看不見的數據的能力。
    • C4.5 是 ID3 的後繼者，並通過動態定義將連續屬性值劃分為一組離散區間的離散屬性（基於數值變量）取消了特徵必須是分類的限制。
    • CART（分類和回歸樹）與 C4.5 非常相似，但不同之處在於它支持數值目標變量（回歸）並且不計算規則集。CART 使用在每個節點產生最大信息增益的特徵和閾值構建二叉樹。
    • scikit-learn 使用 CART 算法的優化版本
  2. Random forest
    • Random forest和K-fold cross-validation 本來就不一樣，一個是model一個是驗證用的方法
    • random forest透過建立多棵decision tree來投票決定最後的結果，而K-fold cross-validation是將資料分為訓練及跟測試集，讓測試的結果跟實際結果對照
    • 也就是說，random forest最後的值終究只是推測，還是需要cross-validation來反覆驗證正確率

With PCA

Principal components analysis (PCA)

Decision Tree

  • 在model.fit()前面call PCA()，將參數設定n_components='mle':此時PCA類會用MLE演算法根據特徵的方差分佈情況自己去選擇一定數量的主成分特徵來降維
  • 把x_train x_test都做transform使訓練跟測試資料的維度一樣






























dtc = DecisionTreeClassifier()

Random Forest

Describe the difference between boosting and bagging.

Bagging
從訓練資料中隨機抽取(取出後放回，n<N)樣本訓練多個分類器，每個分類器的權重一致最後用投票方式得到最終結果

Boosting
和Bagging不同的是分類器之間是有關聯性的，是透過將舊分類器的錯誤資料權重提高，然後再訓練新的分類器

差別

  • 樣本選擇
    • Bagging採用Bootstrap隨機放回抽樣
    • 每輪的training data不變，只有樣本權重會變
  • 樣本權重
    • Bagging採用均勻取樣，使每個樣本權重相等
    • Boosting會根據錯誤率回調權重，錯誤率越大權重越大
  • 預測函式
    • Bagging的預測函式權重相等
    • Boosting中誤差越小的預測函式其權重越大
  • 平行計算
    • Bagging各個預測函式可以並行生成
    • Boosting各個預測函式必須按順序迭代生成
