  • 自定義function
    • euclid_dis(a,b)
      • 計算a,b兩個點之間的距離，用平方公式計算
    • random_centroids(x,k)
      • 首先用np.zeros初始化centroid陣列，開k個row和feature總數的column，紀錄在第k個cluster時的位置
      • index是從train_num裡隨機挑選，作為初始中心點









def euclid_dis(a,b):
    return np.sqrt(np.sum((np.array(a)-np.array(b))**2))
def random_centroids(x,k):
    centroids_ini=np.zeros((k,feature_num))
    index=[x for x in range(train_num)]
    for i in range(k):
        c_id=int(index[i])
        centroids_ini[i]=x[c_id]
    return centroids_ini
  • 自定義function
    • Kmeans(x,k)
      • clusters開train_num(樣本個數)的row和column=2的陣列，第一行紀錄每個sample屬於的cluster，第二行紀錄離其中心點的距離
      • flag用來記錄centroids是否改變(if False -> 中心點已經收斂)
      • while迴圈裡先遍歷每個sample，計算與每個中心點的距離，記錄在distance裡，並找最小距離的值和index(centroid所在的cluster)
      • 找到後再跟原本的sample的centroid比較有沒有一樣，如果不一樣則更新該sample的cluster類別
      • 最後根據新的clusters，分別在各自的類別裡計算features平均值，更新centroid
      • 其他詳細寫在註解上













































def Kmeans(x,k):
    
    #initiate centroids
    centroids=random_centroids(x,k)
    
    #0:屬於哪個centroid 1:distance
    clusters=np.zeros((train_num,2))
    clusters[:,0]=-1
    
    #跑到centroids不再改變為止
    flag=True
    
    while True:
        flag=False
        
        #assignment
        for i in range(train_num):
            distances=[]
            min_dis=1000000.0
            min_index=-1
            #找每個sample最近的centroid
            for j in range(k):
                distance=euclid_dis(x[i,:],centroids[j])
                if distance < min_dis:
                    min_index=j
                    min_dis=distance
            
            #找x在原本cluster裡被歸類的centroid，如果有改變就set flag=True ->下輪繼續跑
            if clusters[i,0]!=min_index:
                clusters[i,:]=min_index,min_dis
                flag=True
        #print("cluster:",clusters)
        
        #update
        for j in range(k):
            #第j個cluster裡的x
            point_in_cluster=x[[m for m in range(train_num) 
                                if clusters[m,0]==j]]
            #採x所有featrues的平均值做為新的centroid
            centroids[j]=np.mean(point_in_cluster,axis=0)
            
        if flag==False:
            break
            
    return centroids,clusters
  • Kmeans結束後根據clusters裡各自x的target label(actual label)投票出最高的label，最多票的即為該cluster裡x的label(predict label)

    • 投票完後更新predict label並記錄到y_predict中與y_true計算accuracy等結果

Comparison & Conclusion

K-means Clustering

  • 因為distance是所有features加起來取平均，取平均下來可能會有與原features的數值有誤差，使判斷不是這麼準確
  • 其中幾個差異特別大的sample也會影響判斷，如原該在cluster 2的點太靠近cluster 1造成誤判

Support Vector Machine

  • 使用三個不同kernel做出來的準確率都不同，因為不同的data維度有各自適合的kernel funciton，其中RBF做出來效果最好
  • 使用 GridSearchCV找到最好的hyperparameter後代入可以發現準確率有很好的提升
