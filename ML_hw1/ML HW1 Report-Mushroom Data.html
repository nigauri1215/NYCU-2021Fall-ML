Data Preprocessing

Drop Features

  • 在stalk-root類別中有些missing value會影響到訓練的準確度故將其移除，另外veil-type資料裡只有’p’的資料沒有’u’，對於決策沒有太大的幫助，也移除


df = m_data.drop(['veil-type'], axis = 1)
df = df[df["stalk-root"] != "?"]

Transform data format and shape

  • 使用Labelencoder來把原本以字符表示的資料轉成整數型表示，以符合model接受的格式





# Using LabelEncoder to convert catergory values to ordinal
labelencoder=LabelEncoder()
for column in df.columns:
    df[column] = labelencoder.fit_transform(df[column])

Shuffle the data

df=sklearn.utils.shuffle(df)

Model Construction & Train-Test-Split

Without Laplace Smoothing

  • 這邊使用sklearn的MultinomialNB
  • alpha為Laplace平滑的常數k，設為1.0e-10，設置夠小讓model足以忽略，相當於alpha=0
nb_without_lap=MultinomialNB(alpha=1.0e-10)

Holdout Validation

  • 將資料集依照7:3分類，7成做為訓練集，3成作為測試集，先把訓練集丟到model裡train，最後用測試集測試準確率
  • 使用sklearn的函數train_test_split，會回傳x y各自的train test，test_size=0.3表示拿30%做test data，剩下做train data





#Holdout validation with the ratio 7:3
x_train,x_test,y_train,y_test=train_test_split(x,y,random_state=42,test_size=0.3)

nb_without_lap.fit(x_train,y_train)
nb_without_lap.score(x_test,y_test)
  • confusion matrix

K-Fold cross-validation

  • 把資料集分為K個子集，用K-1個作為訓練集，剩下1個為測試集，這裡K=3，相當於67% training，33% testing
  • 使用Kfold module，n_split=3表示把資料分為三塊，當shuffle=True時會先打亂原始資料
  • 生成data set之後用for迴圈訓練，每一圈跑一個data set，每圈都會記錄accuracy recall precision和confusion matrix，最後會取平均紀錄到table上















from sklearn.model_selection import KFold
kf=KFold(n_splits=3, random_state=None, shuffle=True)

for train_index, test_index in kf.split(x):
    x_train, x_test = x[train_index], x[test_index]
    y_train, y_test = y[train_index], y[test_index]
    nb_without_lap.fit(x_train,y_train)
    matrix_list.append(confusion_matrix(y_test, nb_without_lap.predict(x_test)))
    kf_acc=accuracy_score(y_test,nb_without_lap.predict(x_test))
    kf_recall=recall_score(y_test,nb_without_lap.predict(x_test))
    kf_prec=precision_score(y_test,nb_without_lap.predict(x_test))
    
    accuracies=nb_without_lap.score(x_test,y_test)

acc_avg=np.mean(accuracies)

Comparison & Conclusion

Without Laplace Smoothing

  • 訓練方法
    • Holdout validation
      • 準確率約86%
    • K-Fold cross-validation
      • 準確率約85%
    • 結果比較
      • 因為兩者方法意義上差不多，都是分割資料集做訓練和測試，且這次Holdout比例是7:3，K-fold的k=3，切割比例是一樣的，所以測試出來的準確度也不會相差太大

With Laplace Smoothing

  • 設置常數alpha=100.0
  • 訓練方法
    • Holdout validation
      • 準確率約85%
    • K-Fold cross-validation
      • 準確率約86%

有無做Laplace smoothing的比較

  • 原本沒有做Laplace的因為某些機率算出來是0，計算的時候就忽略了這筆資料，做平滑就後可以考慮到機率為0的資料
  • 但是在這次實驗中可以發現準確度變化沒有太大，甚至有降低，所以可以發現Laplace smoothing不一定保證提高準確率

Question

Without Laplace Smoothing

  • 首先將label='e’且column='stalk_color_below_ring’的data挑選出來，使用的是dataframe的loc函數，可以篩選特定的column和row
  • 挑好後再用value count計算value frequency，且因為會有value=0的情況，所以使用reindex函數把value=0的項目也讀出來
  • 接著就能計算出在label=e的情況下stalk color below ring各自類別的機率
  • 最後使用matplotlib的pyplot函數畫圖，形式指定bar chart















df=m_data
xi=df.loc[df.label=='e','stalk_color_below_ring'].value_counts().reindex(df.stalk_color_below_ring.unique(), fill_value=0)
print(xi)
y=len(df.loc[df.label=='e'].index)

without_la_prob=round(xi/y,4)
print(without_la_prob)
without_la_prob.plot(kind='bar')
plt.xticks(rotation=0)
plt.xlabel('stalk_color_below_ring')
plt.ylabel('Probability')

for x, y in enumerate(without_la_prob):
    plt.text(x, y, '%s' %y, ha = 'center', va= 'bottom')
plt.show()

With Laplace Smoothing

  • 與上面相同，這次加上alpha=1.0，在分子加上alpha，分母加alpha*feature總個數，即為laplace smoothing

















alpha=1
df=m_data
xi_la=df.loc[df.label=='e','stalk_color_below_ring'].value_counts().reindex(df.stalk_color_below_ring.unique(), fill_value=0)

y=len(df.loc[df.label=='e'].index)

with_la_prob=round((xi+alpha)/(y+alpha*len(df.stalk_color_below_ring.unique())),4)
print(with_la_prob)
with_la_prob.plot(kind='bar')
plt.xticks(rotation=0)
plt.xlabel('stalk_color_below_ring')
plt.ylabel('Probability')

for x, y in enumerate(with_la_prob):
    plt.text(x, y, '%s' %y, ha = 'center', va= 'bottom')

plt.show()
